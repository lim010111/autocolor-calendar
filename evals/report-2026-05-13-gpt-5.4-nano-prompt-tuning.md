- **gpt-5.4-nano default reasoning_effort ≈ `none` (empirical):** C0a 의 `mean_reasoning_tokens = 0.0`, `p95 = 0` (4 lang × 192 = 768 cases). 핸드오프 §7 의 open question 2 — *"gpt-5.4-nano default reasoning_effort 는 무엇인가"* — 가 본 측정으로 답해진다: production default 는 reasoning 을 **실질적으로 사용하지 않는다**. 이는 cost 축의 **floor-bound** 를 의미 — 프롬프트 변경으로 reasoning_tokens 를 더 줄일 여지가 0.
- **API constraint — `minimal` 은 gpt-5.4-nano 에서 미지원:** 핸드오프 §4.2 의 "C0b / C1b minimal 교차" 는 OpenAI API 가 400 으로 거부 (*"Supported values: 'none', 'low', 'medium', 'high', 'xhigh'"*). gpt-5-nano 에는 `minimal` 이 있지만 gpt-5.4-nano 는 없음. - **L2 (action/report separation) — null/슬라이트 음:** 4 lang 평균 Δ ≈ −0.4 %p. en +1.6, ko −0.6, zh-CN −1.6, zh-TW −1.0. nondeterminism 노이즈 (±2 %p) 안에 모두 들어가지만 4 lang 동시 통과는 zh-CN 한 곳에서 0.6 %p 차로 떨어졌다.

- **L5 (literal-first) — uniform degradation:** 4 lang 모두 −1.0 ~ −3.6 %p. V2 의 paraphrase/hypernym/morphology 규칙이 **실제로 일하고 있음** 의 정량 증거.
- **V2 historical reconciliation:** 2026-05-11 V2 historical 은 90.1 / 88.5 / 89.1 / 89.1, 2026-05-13 C0a 는 88.5 / 89.1 / 87.5 / 88.0. 4 lang 평균 Δ = −0.95 %p. nondeterminism (`evals/report.md` §6.1 의 ±1.0–2.1 %p per language) 안에 들어가지만 4 lang 평균이 같은 방향으로 일관되게 떨어지는 점은 *"model snapshot drift 가 있을 수 있다"* 의 약한 시그널. ## 3. Winner Gate 적용 트레이스 (§4.6)

핸드오프 §4.6 *"Winner gate — γ Quality + γ Cost-Pareto"* 를 그대로 적용. **4 lang 모두에서 동시 만족** 해야 하는 simultaneous-language rule.

C0a (V2 same-day baseline) 기준치:
- accuracy = en 0.885, ko 0.891, zh-CN 0.875, zh-TW 0.880
- mean_reasoning_tokens = 0.0 (4 lang 모두)

따라서 thresholds:
- Q1 (`≥ V2 + 2 %p`): en ≥ 0.905, ko ≥ 0.911, zh-CN ≥ 0.895, zh-TW ≥ 0.900
- γ1 C1 (`≥ V2 − 1 %p`): en ≥ 0.875, ko ≥ 0.881, zh-CN ≥ 0.865, zh-TW ≥ 0.870
- γ1 C2 (`mean_r ≤ V2 × 0.7`): **0 × 0.7 = 0** — 모든 셀의 mean_r 도 0 이므로 trivially 통과 (cost 축 floor-bound)

## 5. 발견 사항 (Findings) — 4 lever directional characterization

### 5.1 L1 (follow-up suppression) — null on en/zh-CN, slight negative on ko/zh-TW

C1a vs C0a 4 lang Δ: **en +1.6, ko −4.2, zh-CN +1.6, zh-TW −3.1**. 4 lang 평균 ≈ −1.0 %p.

**해석:** `response_format=json_schema strict` 가 schema-level 로 출력 형식 + termination 을 이미 강제. *"Suppress follow-up questions ... Output only the JSON object, then stop."* 한 줄의 incremental information value 는 en/zh-CN 에서 0 (노이즈 범위), ko/zh-TW 에서는 reasoning chain 을 **방해**하는 부작용. CJK + 한자 시스템에서 gpt-5.4-nano 의 instruction-token sensitivity 가 더 높다는 가설 (검증 미실시).

**Production 함의:** V2 에 본 문장을 추가할 이유 없음. JSON schema strict 가 이미 contract.

### 5.2 L2 (action/report separation) — null/마진

C2 vs C0a: **en +1.6, ko −0.6, zh-CN −1.6, zh-TW −1.0**. 4 lang 평균 ≈ −0.4 %p.

**해석:** `internal:/output:` 라벨이 step 구조에 information value 를 추가하지 않음. en 은 노이즈 +1.6, CJK 는 노이즈 가까운 −1 %p. zh-CN 0.859 가 C1 gate 0.865 를 0.6 %p 차로 떨어진 점은 nondeterminism 의 한계선 — 재실험으로 reversible 할 가능성 있음 (그러나 reversible 마진이 production-ship 의 정량 가치를 만들지 못함).

**Production 함의:** V2 의 `# Exact step order` 의 *plain language* 표기가 *internal/output 라벨* 표기보다 같거나 약간 낫다.

### 5.3 L4 (one cross-lingual example) — surprising mixed-positive

C4 vs C0a: **en +2.6, ko −1.6, zh-CN −1.0, zh-TW +2.1**. 4 lang 평균 ≈ +0.5 %p.

**해석:** *"V2 의 6 개 예시 중 5 개가 noise 일 수 있다"* 가설의 직접 시그널. 단일 cross-lingual KO→EN demonstrator 가 en/zh-TW 에서 V2 의 6-example 합산보다 **잘** 작동. 그러나 ko / zh-CN 은 한 example 의 source language (KO) 와 target language (EN) 가 모두 ZH 가 아니므로 ZH-side demonstrator 손실의 영향이 −1 ~ −1.6 %p 로 나타난다.

**다음 실험 trigger (§6 open Q1):** *"two examples (KO + ZH cross-lingual)"* 변형. 본 측정에서 직접 답하지 못한 *"6 → 1 이 너무 큰 stride 였는가, 아니면 6 → 2 가 sweet spot 인가"* 의 정량 답. 본 PR 의 terminal 결정에도 불구하고 본 lever 는 후속 실험 가치가 가장 높음.

**Production 함의:** V2 의 6 예시는 *"top of pareto"* 가 아닐 수 있다. V2 의 4 ~ 5 예시 또는 2 예시 정도가 더 나을 가능성 — 그러나 4-lang simultaneous 룰을 만족시키려면 추가 측정 필요.

### 5.4 L5 (literal-first) — uniform negative; V2 semantic rules confirmed load-bearing

C5 vs C0a: **en −1.0, ko −2.1, zh-CN −2.6, zh-TW −3.6**. 4 lang 모두 음, 평균 ≈ −2.3 %p.

**해석:** V2 의 *"four ways meaning can match"* 열거 (paraphrase, hypernym, morphology, cross-lingual) 가 **실제로 일하고 있음** 의 정량 증거. 본 규칙들 제거 시 ko/zh-CN/zh-TW 의 morphology + paraphrase 매칭에서 −2 ~ −3.6 %p 의 일관된 하락. zh-TW (繁體) 가 가장 큰 손실 — 번체에서 literal-only 매칭이 가장 손해.

**Production 함의:** V2 §"Critical rule" 의 paraphrase/hypernym/morphology 규칙은 **production critical** — 이후 V2 의 simplification PR 에서 본 규칙들을 건드릴 수 없다.

### 5.5 `reasoning_effort` axis — gpt-5.4-nano default ≈ `none` (empirical answer)

C0a 의 `mean_reasoning_tokens = 0.0 / p95 = 0 / max = 0` (4 lang × 192 cases). gpt-5.4-nano 의 production default 는 reasoning 을 **실질적으로 사용하지 않는다**.

**핸드오프 §7 open question 2 의 답:** default ≈ `none`. C0b 의 의도된 sanity 측정 (`minimal` 으로 cross-check) 은 OpenAI API 가 gpt-5.4-nano + `minimal` 조합을 400 으로 거부하기 때문에 **측정 불가**. 그러나 C0a 의 reasoning_tokens=0 분포가 이미 같은 정보를 제공.

**`minimal` 의 모델별 지원:**
- `gpt-5-nano`: ✓ supported (Wave 1, Stage 1 사용)
- `gpt-5.4-nano`: ✗ unsupported (본 발견)
- Supported on gpt-5.4-nano: `none / low / medium / high / xhigh`

**Production 함의:**
- `LLM_MAX_COMPLETION_TOKENS = 64` 의 안전 마진은 *"happy coincidence"* 가 아니다 — gpt-5.4-nano default 가 reasoning_tokens 를 0 으로 produce 하므로 cap=64 가 항상 충분.
- 프롬프트 차원에서 cost 축의 lever 가 **존재하지 않음** — reasoning_tokens 가 이미 floor. 핸드오프 §4.6 γ Cost-Pareto 게이트의 정량 가치 0 이 본 측정으로 확정.
- ADR-0002 / ADR-0003 의 모델 contract index 에 *"supported reasoning_effort values"* 컬럼 추가 권고 — §6 open Q2.

## 6. Open questions

본 보고서가 직접 답하지 않은 채로 남는 질문 — 다음 prompt 차원 실험의 starting hypotheses.

1. **L4 "two examples (KO + ZH cross-lingual)" 변형의 효과:** C4 가 en/zh-TW 에서 V2 보다 +2 %p 이상 win 했고 ko/zh-CN 에서 −1 ~ −1.6 %p 떨어졌다. 단일 KO→EN demonstrator 가 ZH 쪽에서 부족했다는 해석이 맞다면 *"KO→EN + ZH→EN 두 개 예시"* 가 4-lang 동시 win 의 sweet spot 가능. 추정 cost: 1 cell × 4 lang × 192 = $0.17. 본 PR 의 후속 실험 (가설: C4-2 = `v5-L4-pair`).
2. **모델별 supported `reasoning_effort` 값을 contract index 에 명시:** ADR-0002 의 모델 lock 본문에 *"gpt-5.4-nano supports: none/low/medium/high/xhigh; gpt-5-nano supports: minimal/low/medium/high/xhigh"* 컬럼 추가. 본 PR 의 §5.5 finding 이 향후 같은 함정 (다른 nano 모델 평가에서 `minimal` 으로 sanity-test) 을 막아준다.
3. **L1 의 CJK-specific 음의 부작용 mechanism:** ko/zh-TW 에서 C1a 가 V2 보다 −3 ~ −4 %p 떨어진 것이 instruction-token sensitivity 인지, 추가 한 줄 자체가 *"의미 없이 한 줄 더 있는"* 문맥 disruption 인지, 다른 mechanism 인지 확인하려면 *"L1 변형: V2 + 무관한 한 줄 (e.g. 날씨 코멘트)"* 을 negative control 로 측정해야 한다. 본 PR 의 priority 는 낮음 (정성 답이 충분히 명확).
4. **C2 zh-CN / C4 ko 0.6 %p 의 nondeterminism 마진:** 두 cell 모두 동일 측정을 반복하면 한쪽이 통과할 가능성이 있다. 4-lang simultaneous robustness 룰은 이를 의도적으로 fail 시키지만, *"두 cell 을 각 3 회 반복 측정한 평균"* 으로 새로 게이트를 그리는 것도 옵션. 추정 cost: 2 cells × 3 runs × 4 langs × 192 = $0.40.
